# v7.9 パーティ枠制限準備

## 目的

v7.6で表示だけ追加していた「1枠 / 2枠 / 3枠」を、実際のパーティ編成判定に使える状態へ進めました。

今後ドラクエモンスターズ寄りの大型モンスター制に近づけるため、パーティ人数ではなく「合計パーティ枠」で管理します。

## 変更内容

- `GAME_VERSION` を `7.9` に更新
- `PARTY_SLOT_LIMIT = 3` を追加
- `MonsterLinksState` に以下のヘルパーを追加
  - `partySlotLimit()`
  - `monsterSize(monsterOrId)`
  - `partySizeUsed(list)`
  - `partySlotsRemaining(list)`
  - `canAddToParty(monster, list)`
  - `partySizeText(list)`
- 仲間画面にパーティ枠ゲージを追加
- 詳細モーダルに、対象モンスターの使用枠と現在のパーティ枠を表示
- 牧場からパーティへ移動する時、合計枠が3を超える場合は移動できないように変更
- スカウトや配合で仲間が増える時、枠が足りない場合は牧場へ送るように変更
- 既存セーブで合計枠が超過している場合、読み込み時に超過分を牧場へ移動

## 今回あえて変更していないこと

- 戦闘の基本仕様
- サイズによる能力補正
- 配合レシピ
- 配合後Lv1仕様
- 引き継ぎ技最大2つ仕様
- 保護ロック仕様

## 確認ポイント

1. 仲間画面で「パーティ枠 〇/3」が表示される
2. 2枠モンスターがいる時、残り枠が正しく表示される
3. 合計3枠を超えるモンスターは牧場からパーティへ入れられない
4. 詳細モーダルで「この仲間：〇枠 / 現在のパーティ：〇/3枠」が表示される
5. 配合後の子が2枠以上で、パーティに入らない場合は牧場へ送られる
