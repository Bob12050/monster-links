# ランク・サイズ制 設計

## ランク案

現在のランクに、将来的にSS以上を追加します。

```text
F
E
D
C
B
A
S
SS
SSS
```

## まず入れるなら

最初はSSまでで十分です。

```text
S: 終盤の主力・通常最上位
SS: 特殊配合・大型・隠し級
SSS: 将来の超高難度・最終目標
```

## サイズ制案

```text
1枠: 通常モンスター
2枠: 大型モンスター
3枠: 超大型モンスター
```

## パーティ枠案

```text
パーティ最大枠: 3
1枠 + 1枠 + 1枠 = OK
2枠 + 1枠 = OK
3枠 = OK
2枠 + 2枠 = NG
3枠 + 1枠 = NG
```

## 1枠モンスター

```text
数が多い
育成しやすい
役割分担しやすい
序盤〜終盤まで使える
```

## 2枠モンスター

```text
HP・攻撃・耐久が高め
強力な全体技や高威力技を持つ
編成自由度は下がる
ボス・大型ドラゴン・大型機械などに向く
```

## 3枠モンスター

```text
1体だけでパーティを占有
SS/SSS級に向く
超大型ボス・神獣・古代兵器向け
圧倒的だが状態異常やMP切れに弱いなど弱点も必要
```

## 実装時に影響する場所

```text
パーティ編成
仲間画面
戦闘画面
図鑑
配合結果
敵出現
開発者モード
バランス調整
```

## 実装方針

いきなり戦闘に反映しない。
まずはデータに `size:1` を持たせるところから始めるのが安全です。

段階案:

```text
1. データだけ size を追加
2. 図鑑・仲間画面にサイズ表示
3. パーティ編成で枠制限
4. 2枠/3枠専用ステータス調整
5. 2枠/3枠専用技を追加
```
