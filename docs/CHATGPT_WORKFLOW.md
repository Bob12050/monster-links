# 普通のChatGPTとの開発運用手順

## 新しいチャットを始める時

添付するもの:

```text
最新版のプロジェクトZIP
docs/PROJECT_SPEC_CHATGPT.md
必要な参考画像
```

送る文章:

```text
docs/CHATGPT_HANDOFF_PROMPT.mdの本文
今回行いたい具体的な作業
```

普通のChatGPTは、このPCのフォルダや前回チャットのファイルを自動では参照できません。
新しいチャットでは必ず最新版ZIPを添付してください。
同名ZIPを更新した場合は、添付前に古い添付を削除し、更新日時またはSHA256も伝えると取り違えを減らせます。

## 1回の作業単位

```text
1. 最新ZIPを渡す
2. やりたいことを1テーマに絞る
3. 変更案を確認する
4. ファイルを変更してもらう
5. 変更ファイル一覧を受け取る
6. 可能なら更新ZIPを受け取る
7. GitHub Pagesで動作確認する
8. 次回は更新後ZIPを渡す
```

## 依頼文テンプレート

```text
添付ZIPが最新版です。
今回は「ここに作業内容を書く」をお願いします。

条件:
- 既存セーブ互換性を維持
- GitHub Pagesで動く静的構成を維持
- 無関係な変更をしない
- 変更ファイルを明示
- 可能なら検証結果を報告
- 更新後のZIPを返す
```

## アート差し替えテンプレート

```text
添付ZIPが最新版です。
添付した確認済み画像を、モンスターID「ID」の正式画像として反映してください。

条件:
- assets/images/monsters/ID.pngへ配置
- 1024x1024背景透過PNG
- 旧画像をlegacyフォルダへ保存
- 名前、ID、能力、配合、セーブ形式は変更しない
- アート管理表とバージョン履歴を更新
- tools/validate.mjsとtools/art-audit.mjsで確認
- 変更ファイル一覧と更新ZIPを返す
```

## 機能追加テンプレート

```text
添付ZIPが最新版です。
「追加したい機能」を実装してください。

実装前に関連するcore、systems、views、CSS、既存テストを確認してください。
セーブ項目を増やす場合は、旧セーブに既定値を補完してください。
画面はスマホ幅を優先し、重要文字を10px未満にしないでください。
```

## ChatGPTから受け取る報告形式

```text
実装内容
変更ファイル
セーブ互換性への影響
検証内容と結果
未確認事項
更新ZIP名
更新ZIPのSHA256
```

## 同時進行を避ける

普通のChatGPTとCodexの両方で同じZIPを同時編集すると、変更が食い違います。

```text
片方の変更を完了
↓
更新ZIPまたは最新フォルダを確定
↓
もう片方へ渡す
```

この順番を守ってください。

並行作業が必要な場合は作業テーマを別ファイル領域に分け、最後に1つの最新版へ統合してから次の作業を始めます。

## 仕様書を更新するタイミング

次の場合は`docs/PROJECT_SPEC_CHATGPT.md`も更新します。

```text
ゲームバージョン変更
モンスター数やステージ数の変更
セーブ形式の変更
主要機能の追加
アート方針の変更
承認済みデザインの追加
開発ロードマップの変更
```

## 引き継ぎで特に確認すること

```text
ZIP内のGAME_VERSION
js/core/monsters.jsのモンスター数
js/core/stages.jsのステージ数
js/core/recipes.jsの配合数
SAVE_SCHEMA_VERSION
承認済みアート一覧
直前に変更したファイル
tools/validate.mjsの結果
```

## アート制作の受け渡し

```text
1. ID、名前、ランク、属性、役割を確認
2. 白背景の確認用イラストを生成
3. ユーザーがデザインを承認
4. 同じデザインで1024x1024背景透過PNGを生成
5. 透明四隅、余白、切れ、文字混入を確認
6. ID.pngとしてゲームへ反映
7. 旧画像をlegacyへ保存
8. art-auditと総合検証を実行
```

現在の創作上の採用済みデザインは10体です。
技術上の`formal_png`件数と混同しないでください。
