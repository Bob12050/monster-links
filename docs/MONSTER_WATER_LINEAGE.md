この文書は、Monster Links の「水・湿地・氷・深海系」モンスターの系統、役割、アート方針、現在のデザイン承認状況を整理するための資料です。

ここでの「採用候補」「承認」は、白背景確認用デザインの方向性承認を意味します。  
透明PNG化、画像ファイル差し替え、表示名変更、ゲーム実装は、ユーザー確認後の別作業として扱います。  
v8.6-A.2 で既に実装済みのものもありますが、後から生成・承認されたものはまだゲームには反映していません。

## 1. 基本方針

水・氷・深海系は、清水・湿地・毒沼・氷雪・海底・深海・水精霊を表現する系統です。

共通方針:

```text
水、泡、波、雫、氷、雪、珊瑚、貝殻、深海発光などを使う
透明感と鮮やかな青系の色を重視する
水系でも役割が被らないように、清水・毒沼・氷・深海・海底守備型を分ける
低ランクは親しみやすく、上位ランクは神秘性・体格・装飾で強さを出す
スマホ画面でも輪郭と特徴が分かる
人型精霊は高ランクに見えすぎないよう注意する
既存の維持対象10体とは被らせない
```

## 3. 重要な保留方針

### タイダルセラフ

直近で生成した人型水精霊案は、以下の理由で保留です。

```text
Bランクにしては豪華すぎる
人型・高ランク感が強い
パールセラフと役割が被る
セレスティアルセラフ級に見える
```

今後の方針:

```text
Bランクの水セラフ/上位水精霊として再設計する
装飾を控えめにする
パール・真珠・神聖感を弱める
水流・潮・波・青い羽を中心にする
人型すぎない方向も検討する
```

---

### しずくぷる

`dewplim / しずくぷる` は、水系ではなくぷる系側の保留枠として扱います。

理由:

```text
新しいぷるミンが青いしずく型になったため存在が被る
単純な水色ぷるとして作ると差別化が弱い
```

今後の候補:

```text
ミストぷるなどに改名する
霧・水蒸気・泉の精霊に方向転換する
別モンスター枠へ置き換える
```

## 4. 水・氷・深海系の中核ライン

### 4.1 清水・水精霊ライン

```text
アクアン
└─ タイダルセラフ（保留・再設計）
   └─ パールセラフ
```

現状では、アクアンは低ランクの清水精霊として良好です。  
タイダルセラフはBランクの中間精霊として再設計が必要です。  
パールセラフはAランクの水・光セラフとして採用候補です。

---

### 4.2 氷雪ライン

```text
ユキまる
├─ スノーフェア
├─ スノーニャ
├─ アイスタートル
└─ フロストリヴァイア
```

注意:

```text
ユキまる = 氷の子犬・雪獣系
スノーニャ = 氷翼の猫・雪豹/リンクス系
```

ユキまるからスノーニャへ「犬から猫に進化」して見えるのは違和感があるため、  
スノーニャは直系進化ではなく、氷・飛行・猫系の別ライン/交差派生として扱います。

---

### 4.3 湿地・毒沼ライン

```text
ミアトード
└─ ヘドロトカゲ
```

ミアトードは自然系でもありますが、水辺・湿地系との交差枠です。  
ヘドロトカゲは毒沼・汚泥・毒液の枝として、清水系とは明確に分けます。

---

### 4.4 深海ライン

```text
アビスジェリー
├─ アビスフィン
└─ アビスリヴァイア
```

深海ラインは、発光・濃紺・神秘性を持つ水系上位枝です。

```text
アビスジェリー = 深海クラゲ
アビスフィン = 深海魚
アビスリヴァイア = 深海竜・Sランク
```

アビスジェリーは青い幻想的な1枚目を採用候補とし、  
後から生成した怖すぎる紫のクラゲ案は不採用です。

---

### 4.5 海底・珊瑚・貝殻ライン

```text
コーラルルーメ
└─ シェルゴーレム
```

コーラルルーメは珊瑚・水・光寄りの綺麗な枝です。  
シェルゴーレムは海底守備型で、貝殻・岩・珊瑚・海藻をまとった大型/中型の防御モンスターとして扱います。

## 7. 実装時の注意

```text
確認済みデザインのみ透過PNG化する
未承認アートをゲームに正式反映しない
画像更新と能力・配合・セーブ形式変更を混ぜない
既存IDを安易に変更しない

画像は assets/images/monsters/{id}.png に配置する
旧画像は必要に応じて legacy フォルダへ退避する
実装後は tools/validate.mjs と tools/art-audit.mjs を実行する
```
