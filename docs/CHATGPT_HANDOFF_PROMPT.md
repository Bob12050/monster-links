# 普通のChatGPTへ渡す引き継ぎプロンプト

以下の本文を、新しいChatGPTの最初のメッセージとして貼り付けてください。
同時に、最新版のプロジェクトZIPと`docs/PROJECT_SPEC_CHATGPT.md`を添付してください。

```text
Monster LinksというGitHub Pages用のHTML/CSS/JavaScript RPGを開発しています。
添付した最新ZIPをベースに、今後も継続して開発を手伝ってください。

最初に次の順で確認してください。

1. 添付ZIPのフォルダ構成
2. docs/PROJECT_SPEC_CHATGPT.md
3. README.md
4. docs/CURRENT_VERSION.md
5. js/core/config.js
6. tools/validate.mjs

重要ルール:

- 添付ZIPの実ファイルを必ず最新版として扱う
- 過去の会話や古い資料より実ファイルを優先する
- GitHub Pagesで動く静的サイトを維持する
- HTML/CSS/JavaScript/JSON/画像のみで構成する
- 外部サーバー、データベース、API、ビルド環境を必須にしない
- ReactやVueなどへの全面移行はしない
- 既存セーブデータ互換性をなるべく壊さない
- モンスターID、配合、能力を無関係な作業で変更しない
- ユーザーが確認していないアートを正式反映しない
- 変更前に関連ファイルを確認する
- 変更後に可能ならtools/validate.mjsを実行する
- 変更したファイルを必ず明示する
- 完成した変更ファイル一式または更新ZIPを返す

プロジェクトの現在方針:

- 現在の安定版はv8.6-A.1
- モンスターは84体
- キャラクターアートを全体的に同じ画風へ作り直す
- 新たに確認済みの高ランク10体は現在のデザインを維持する
- ぷるミンは、ユーザーが提示する青いしずく型デザインへ変更予定
- それ以外の既存キャラクターは順次再デザインする
- ゲーム機能とセーブデータはアート変更と分離する
- UIはスマホの可読性と操作性を優先する

回答は日本語でお願いします。

まずZIPと仕様書を読み、以下だけを簡潔に報告してください。

1. 認識した現在バージョン
2. プロジェクト構成
3. セーブ互換性で注意する点
4. 現在のアート方針
5. 次に行うべき作業候補

私が作業を指定するまでは、まだファイルを変更しないでください。
```

## すぐ作業も依頼する場合

上の最後の一文を削除し、代わりに次を追加してください。

```text
今回の依頼:
[ここへ具体的な作業を書く]

関連コードを先に確認したうえで、提案だけで止めず、可能な範囲で実装、検証、更新ZIP作成まで行ってください。
判断できない点だけ質問し、それ以外は既存実装に合う保守的な方法を選んでください。
```

アート方針の「確認済み」は、画像ファイルの形式ではなくユーザーのデザイン承認を意味します。
`formal_png`という管理状態だけで採用済みと判断しないでください。

## アート制作だけを依頼する場合

最新版ZIPの代わりに、次を添付してください。

```text
docs/PROJECT_SPEC_CHATGPT.md
ぷるミンの基準画像
採用済み高ランクモンスター2〜3枚
作成対象モンスターの名前とID
```

追加メッセージ:

```text
今回はコードを変更せず、確認用イラストと生成プロンプトだけを作ってください。
白背景版を先に提示し、私が承認するまで透過PNG化やゲーム反映を行わないでください。
```
